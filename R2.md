# 代码变更说明 (Changelog)

## 概述

本次更新涉及 4 个文件：`train_qwen.py`、`trainer.py`、`argument.py`、`sft.sh`，核心目标是：
1. 支持 LoRA 微调 LLM，大幅降低显存占用
2. 修复 DeepSpeed CPU offload 导致的 OOM 问题
3. 训练结束后在 GPU 上直接合并模型，无需二次加载
4. 增强 checkpoint 校验逻辑，防止 OOM 中断后残缺 checkpoint 误判

---

## `argument.py`

### 新增：LoRA 相关参数字段

`ModelArguments` 中新增 5 个字段：

```python
# 新增
use_lora: bool = field(default=False)
lora_r: int = field(default=64)
lora_alpha: int = field(default=128)
lora_dropout: float = field(default=0.05)
lora_target_modules: str = field(
    default="q_proj,k_proj,v_proj,o_proj,gate_proj,up_proj,down_proj"
)
```

旧版 `ModelArguments` 仅含 `model_name_or_path`、`tune_mm_llm`、`tune_mm_mlp`、`tune_mm_vision` 四个字段，不支持 LoRA。

## `train_qwen.py`

### 1. 新增 `GradientDebugCallback`

训练开始时记录所有可训练参数的初始值，第 3 步后打印参数变化量，用于快速验证梯度是否正常流动。

```
[GradCheck] ✅ lora_A.weight: max_diff=3.81e-06   ← 正常更新
[GradCheck] ❌ merger.norm.weight: max_diff=0.00   ← 未更新（预期行为）
```

旧版无任何训练过程中的梯度诊断。

### 2. `get_visual_module` 兼容性增强

新版增加对 PEFT 包装层（`base_model`）和 DeepSpeed 包装层（`module`）的逐层解包，确保 LoRA wrap 后仍能正确找到视觉模块。

```python
# 新版逐层解包
if hasattr(m, 'base_model'): m = m.base_model
if hasattr(m, 'model'):      m = m.model
if hasattr(m, 'module'):     m = m.module
```

旧版仅做两层判断，LoRA wrap 后会报 `AttributeError`。

### 3. `set_model` 执行顺序调整

| 旧版顺序 | 新版顺序 |
|---------|---------|
| ① 视觉编码器 | ① LLM |
| ② Merger | ② 视觉编码器 |
| ③ LLM | ③ Merger（最后执行，优先级最高） |

新版将 merger 放在最后设置，确保 PEFT wrap 或其他操作不会意外覆盖 merger 的 `requires_grad`。同时新增 DEBUG 打印输出各模块训练状态。

### 4. 新增 `apply_lora` 函数

使用 `peft` 库对 LLM 部分应用 LoRA，wrap 完成后显式恢复 merger 的 `requires_grad`（PEFT 可能将非 LoRA 参数重置为不可训练）。

```python
def apply_lora(model_args, model):
    lora_config = LoraConfig(r=64, lora_alpha=128, target_modules=[...])
    model = get_peft_model(model, lora_config)
    # 恢复 merger requires_grad
    for p in merger.parameters():
        p.requires_grad = True
    return model
```

旧版不支持 LoRA，全量微调 LLM 需要 80GB+ 显存。

### 5. `enable_input_require_grads` 无条件调用

旧版仅在 `gradient_checkpointing=True` 时注册 hook；新版无条件调用，确保存在冻结层时计算图仍然连通。

```python
# 旧版（条件调用）
if training_args.gradient_checkpointing:
    model.enable_input_require_grads()

# 新版（无条件调用）
model.enable_input_require_grads()
if training_args.gradient_checkpointing:
    model.gradient_checkpointing_enable()
```

### 6. Checkpoint 校验逻辑增强

旧版只检查 `checkpoint-*` 目录是否存在，OOM 中断后的残缺 checkpoint 会被误判为有效而导致 `FileNotFoundError: scheduler.pt`。

新版额外校验三个文件均存在才认定为有效：

```python
def is_valid_checkpoint(ckpt_path):
    has_state     = (ckpt_path / "trainer_state.json").exists()
    has_ds_latest = (ckpt_path / "latest").exists()
    has_scheduler = (ckpt_path / "scheduler.pt").exists()  # 新增
    return has_state and has_ds_latest and has_scheduler
```

### 7. 训练结束保存策略重构

| 旧版 | 新版 |
|------|------|
| `safe_save_model_for_hf_trainer`（DeepSpeed 同步保存完整权重，内存峰值高） | ① 保存 LoRA adapter（体积小，备用） |
| | ② GPU 上原地 `merge_and_unload`（无需二次加载） |
| | ③ 保存完整 merged_model（可直接推理） |

```
checkpoints/qwen3vl_caption/
├── lora_adapter/          ← LoRA adapter 备份
└── merged_model/          ← 完整模型，直接用于推理
```

### 8. 新增可训练参数汇总打印

```
[INFO] ══════ 可训练参数汇总 ══════
  LoRA 参数:    150.0M (504 tensors)
  Merger 参数:  24.0M  (24 tensors)
  合计:         174.0M
```

## `trainer.py`

### 1. `create_optimizer` 新增空参数组过滤

旧版在 LoRA 模式下某些参数组可能为空，传给 DeepSpeed 会触发 `IndexError`。新版在构建 optimizer 前过滤：

```python
# 新增
optimizer_grouped_parameters = [
    g for g in optimizer_grouped_parameters if len(g["params"]) > 0
]
```

### 2. 新增 `save_model` 覆盖

旧版使用 `safe_save_model_for_hf_trainer` 保存完整模型（CPU 峰值内存 ~50GB，容易 OOM）。新版覆盖 `Trainer.save_model`，只保存可训练参数到 `merger_weights.pt`：

```python
TRAINABLE_KEYWORDS = ["merger", "patch_merger", "deepstack_merger", "lora_"]

def save_model(self, output_dir=None, _internal_call=False):
    # 只保存关键字匹配的可训练参数
    torch.save(trainable_state_dict, "merger_weights.pt")
```

### 3. 新增 `_save_checkpoint` 覆盖

旧版每次 checkpoint 保存 8B 完整权重，磁盘和内存压力大。新版：
- 只保存可训练参数（`merger_weights.pt`）
- 调用 `deepspeed.save_checkpoint(exclude_frozen_parameters=True)` 跳过冻结的 LLM 权重
- 自动管理 checkpoint 数量上限（`save_total_limit`）

## `sft.sh`

| 参数 | 旧版 | 新版 | 原因 |
|------|------|------|------|
| 模型 | Qwen3-VL-**7B** | Qwen3-VL-**8B** | 使用更新的模型 |
| `--use_lora` | ❌ 无 | ✅ 启用 | 降低显存至 ~35GB |
| `--lora_r` | ❌ 无 | `64` | LoRA rank |
| `--lora_alpha` | ❌ 无 | `128` | LoRA scaling |
| `--lora_dropout` | ❌ 无 | `0.05` | 防过拟合 |
| `--learning_rate` | `2e-6` | `1e-5` | LoRA 模式下 LR 可适当提高 |
| `--mm_projector_lr` | `1e-5` | `2e-5` | Merger 独立学习率提高 |
| `--max_pixels` | `384×384×3` | `512×512×3` | 支持更高分辨率卫星图 |
| `--model_max_length` | `2048` | `4096` | 6张图 + 长文本描述需要更长序列 |
| DeepSpeed | ZeRO-2 + **CPU offload**（触发 OOM） | ZeRO-2 **无 CPU offload** | 单卡 A800+LoRA 无需 offload |
| `--dataloader_num_workers` | `4` | `2` | 减少 RAM 压力 |
| `--run_name` | `qwen3vl_caption_finetune` | `qwen3vl_lora_finetune` | 名称与配置对应 |
